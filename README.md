# تحدي كأس العالم ٢٠٢٦ — الحسني هوم سنتر

منصة داخلية لتوقعات الموظفين على المباريات الثمان المتبقية من كأس العالم (ربع النهائي → النهائي)، بنقاط حيّة، ترتيب فردي وفرعي، لوحة إدارة، وحفل تتويج.

## التشغيل

```bash
npm install
npm run build        # يترجم TypeScript → public/js
npm run seed         # المباريات الحقيقية + حساب الإدارة (أضف --demo لثلاثة حسابات تجريبية)
npm start            # http://localhost:3000
```

**حساب الإدارة:** `laethalkawaz / laethalkawaz` — غيّر كلمة المرور فور أول دخول (الموظفون ← كلمة مرور جديدة).

## الإعدادات (.env)

```bash
cp .env.example .env
```
يُقرأ الملف تلقائياً عند الإقلاع (Node native — بلا dotenv). متغيرات النظام لها الأولوية.

| المتغير | الافتراضي | الوظيفة |
|---|---|---|
| `PORT` | 3000 | منفذ التطبيق |
| `NODE_ENV` | production | `development` يُظهر تفاصيل الأخطاء للاستجابة |
| `COOKIE_SECURE` | 0 | فعّلها خلف HTTPS |
| `TRUST_PROXY` | 1 | قراءة IP الحقيقي خلف Nginx لسجل التدقيق |
| `DB_PATH` | data/worldcup.db | مسار قاعدة البيانات |
| `JWT_SECRET` | يولَّد ويُحفظ تلقائياً | حدّده فقط للتدوير أو تعدد النسخ |

## إضافة الموظفين
لوحة الإدارة ← الموظفون ← استيراد قائمة، سطر لكل موظف:
```
username,الاسم الكامل,الفرع,القسم
ali.h,علي حسين,المنصور,المبيعات
```
كلمات المرور تُولَّد تلقائياً وتُنزَّل كملف CSV يفتح بـ Excel.

## نظام النقاط
| | نقاط |
|---|---|
| نتيجة دقيقة (٩٠ دقيقة) | ٥ |
| اتجاه صحيح (فوز/تعادل) | ٢ |
| توقع المتأهل (بعد الترجيح) | +٢ |
| مكافأة توقع البطل (يُقفل قبل أول ربع نهائي) | +١٠ |

المضاعفات: ربع النهائي ×١ · نصف النهائي والبرونزية ×٢ · النهائي ×٣ · جوكر ×٢ (مرة واحدة).
كسر التعادل: عدد النتائج الدقيقة ثم أسبقية وقت التوقع.

## سير العمل يوم المباراة
1. التوقعات تُقفل تلقائياً عند صافرة البداية (لا حاجة لأي تدخل).
2. بعد النهاية: الإدارة ← النتائج ← أدخل النتيجة (+المتأهل عند التعادل) ← اعتماد.
3. النقاط والترتيب والإنجازات تُحتسب وتُبث حياً لكل الشاشات المفتوحة (SSE)، وشبكة الأدوار تتحدّث تلقائياً (الفائز ينتقل لنصف النهائي، الخاسر للبرونزية).
4. بعد النهائي: صفحة `/ceremony.html` تفتح للجميع — وزر «وضع العرض للإدارة» لشاشة القاعة.

## النشر خلف Nginx

```nginx
server {
    listen 80;
    server_name worldcup.alhasani.iq;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        # SSE
        proxy_http_version 1.1;
        proxy_set_header Connection '';
        proxy_buffering off;
        proxy_read_timeout 24h;
    }
}
```

### خدمة systemd
```ini
[Unit]
Description=Al-Hasani World Cup Challenge
After=network.target

[Service]
WorkingDirectory=/opt/worldcup
ExecStart=/usr/bin/node server/index.js
# الإعدادات تُقرأ تلقائياً من /opt/worldcup/.env
Restart=always
User=www-data

[Install]
WantedBy=multi-user.target
```

قاعدة البيانات ملف واحد `data/worldcup.db` (SQLite WAL) — النسخ الاحتياطي = نسخ الملف.

## فرادة أسماء الموظفين (هجرة تلقائية)
عند أول إقلاع بعد هذا الإصدار يُنشأ قيد فريد على اسم الموظف (`UNIQUE` على مستوى القاعدة، غير حساس لحالة الأحرف ومع تجاهل الفراغات الطرفية). إذا كانت قاعدة قديمة تحوي أسماء مكررة: **تتوقف الهجرة ولا يُعدَّل أي سجل** — يُطبع تقرير بالسجلات المتعارضة (المعرف واسم المستخدم لكل تكرار) مع أمر مثال للتصحيح، ثم أعد التشغيل. `username` يبقى فريداً كما هو.
